UMP-ST

The Uncertainty Modeling Process for Semantic Technologies (UMP-ST) was proposed by Rommel Carvalho as a methodology for modeling Probabilistic Ontologies. It is based on the Unified Process (UP), building the ontology incrementally and iteratively.

The basic idea behind iterative enhancement is to model our domain incrementally, allowing the modeler to take advantage of what was being learned during the modeling of earlier, incremental, deliverable versions of the model.


Based on UP, the UMP-ST have four phases:
  1. Inception: the goals are defined
  2. Elaboration: the ontology is modeled
  3. Construction: the ontology is implemented
  4. Transition: a new version of the ontology will be available

The UMP-ST consists of four major disciplines:

  • Requirements: the user define the goals that should be achieved
  • Analysis & Design: efetive modeling, were the user will define the entities (with their attributes and relationships), rules and groups that will compose the domain.
  • Implementation: the user will implement the probabilistic ontology using one of the technologies available
  • Test: the user test if the PO has the extepcted behaviour.
